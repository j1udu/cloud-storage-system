# 前端开发日志 - 2026/04/25

## 本次完成

### 1. Vue 3 前端工程初始化
- 新增 `frontend` 独立前端工程，技术栈为 Vue 3 + TypeScript + Vite。
- 接入 Pinia 作为登录态与用户信息状态管理方案。
- 接入 Vue Router，完成 `/login`、`/files` 与兜底重定向路由。
- 接入 Element Plus 与图标库，作为表单、表格、弹窗、分页、消息提示等基础 UI 能力。

### 2. 登录与注册页面
- 实现登录/注册双模式切换。
- 增加用户名、密码、昵称的前端表单校验。
- 登录成功后持久化 token、过期时间与用户信息，并根据 redirect 参数回跳目标页面。
- 注册成功后回到登录模式，方便继续登录。

### 3. 鉴权与请求封装
- 封装统一 `request` 方法，自动拼接 `VITE_API_BASE_URL`。
- 自动为非 `FormData` 请求补充 `Content-Type: application/json`。
- 自动从本地存储读取 token，并注入 `Authorization: Bearer <token>`。
- 统一处理后端业务错误码；token 失效时清理本地登录态并跳转登录页。

### 4. 文件管理页面
- 实现文件列表查询，支持当前目录、分页、刷新。
- 实现文件上传，使用 `multipart/form-data` 与后端 `/files/upload` 对接。
- 实现文件下载，通过后端返回的预签名 URL 打开下载链接。
- 实现文件/文件夹重命名。
- 实现软删除操作，并带确认弹窗。
- 实现面包屑目录导航，为后续文件夹能力预留交互入口。

### 5. 工程化与运行文档
- 新增 `frontend/.env.example`，说明前端 API 基础路径配置。
- 新增 `frontend/README.md`，记录本地开发命令与已接入接口。
- 更新 `.gitignore`，忽略前端依赖目录、构建产物、TypeScript 构建缓存和本地环境变量。

## 验证结果

```bash
npm run build
```

构建通过，已完成 TypeScript 类型检查与 Vite 生产构建。

构建时 Vite 提示主 chunk 超过 500 kB，这是 Element Plus 全量样式与依赖体积带来的优化提醒，不影响当前功能运行。后续可以通过组件按需引入、路由级拆包或 `manualChunks` 继续优化。

## 遇到的问题

1. 沙箱内运行前端构建时，Vite 启动 esbuild 子进程被系统拒绝并报 `spawn EPERM`。
2. 在获得权限后于沙箱外重新执行同一构建命令，构建通过。
