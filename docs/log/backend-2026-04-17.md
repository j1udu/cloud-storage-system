# 后端开发日志 - 2026/04/17

## 今日完成

### 1. 数据库精简重设计
- 将 5 张表精简为 2 张（users + matter），聚焦核心功能
- matter 表用 `dir` 字段合并文件和文件夹
- 分片上传、分享等功能后续按需加表

### 2. 用户模型定义（model/user.go）
- 定义 User 结构体对应数据库表
- 定义 RegisterRequest、LoginRequest、LoginResponse 请求响应结构
- 使用 `json:"-"` 标签防止密码暴露给前端

### 3. 密码加密工具（pkg/hash/password.go）
- 使用 bcrypt 算法加密密码
- 实现哈希生成（HashPassword）和验证（CheckPassword）两个函数

### 4. JWT 令牌工具（pkg/jwt/jwt.go）
- 实现令牌生成（GenerateToken）和解析验证（ParseToken）
- 使用 HS256 签名算法，支持自定义过期时间
- Claims 中存储 UserID 和标准字段（exp、iat、iss）

### 5. 统一响应格式（handler/response.go）
- 封装 Success 和 Fail 两个响应函数
- 所有接口返回统一格式：{code, msg, data}

### 6. 错误码定义（pkg/errcode/errcode.go）
- 定义 5 个业务错误码（10001-10005）
- 实现 GetMsg 函数将错误码映射为中文消息

### 7. 用户数据访问层（repository/user_repo.go）
- 实现 Create、GetByUsername、GetByID 三个数据库操作
- 使用依赖注入接收数据库连接池

### 8. 用户业务逻辑层（service/user_service.go）
- 实现注册流程：检查重名 → 加密密码 → 存入数据库
- 实现登录流程：查用户 → 验证密码 → 生成 JWT
- 实现获取用户信息：按 ID 查询
